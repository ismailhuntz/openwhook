Implementation Guide

Send any HTTP request to your webhook URL and it will be captured in real-time. Below are examples in multiple languages showing how to send webhooks to OpenWhook.

How it works

1
Create a session — visit the app to get a unique webhook URL like https://yourhost.com/hook/<session-id>
2
Send HTTP requests — POST, GET, PUT, or any method to that URL from your app, cURL, or any HTTP client
3
Inspect in real-time — headers, body, query params, IP, and timestamps appear instantly via WebSocket

Code Examples

Choose your language. Each example sends a JSON payload to your webhook URL.

POST request with JSON body 
# Send a POST request with JSON payload
curl -X POST WEBHOOK_URL \
  -H "Content-Type: application/json" \
  -d '{"event": "order.created", "data": {"id": 123, "total": 49.99}}'
GET request with query params 
# Send a GET request with query parameters
curl WEBHOOK_URL"?event=ping&source=monitoring"
PUT request with custom headers 
# Send a PUT request with auth header
curl -X PUT WEBHOOK_URL/api/update \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer your-token-here" \
  -d '{"status": "active", "updated_at": "2026-01-15T10:30:00Z"}'
Form data submission 
# Send form-encoded data
curl -X POST WEBHOOK_URL \
  -d "name=John+Doe&email=john@example.com&plan=pro"

net/http 
package main

import (
	"bytes"
	"encoding/json"
	"fmt"
	"net/http"
)

func main() {
	webhookURL := "WEBHOOK_URL"

	payload := map[string]any{
		"event": "order.created",
		"data": map[string]any{
			"id":    123,
			"total": 49.99,
		},
	}

	body, _ := json.Marshal(payload)
	resp, err := http.Post(webhookURL, "application/json", bytes.NewReader(body))
	if err != nil {
		panic(err)
	}
	defer resp.Body.Close()

	fmt.Println("Status:", resp.Status)
}

HttpClient (Java 11+) 
import java.net.URI;
import java.net.http.HttpClient;
import java.net.http.HttpRequest;
import java.net.http.HttpResponse;

public class WebhookExample {
    public static void main(String[] args) throws Exception {
        String webhookUrl = "WEBHOOK_URL";

        String json = """
            {
                "event": "order.created",
                "data": {
                    "id": 123,
                    "customer": "jane@example.com",
                    "total": 49.99
                }
            }
            """;

        HttpClient client = HttpClient.newHttpClient();
        HttpRequest request = HttpRequest.newBuilder()
            .uri(URI.create(webhookUrl))
            .header("Content-Type", "application/json")
            .POST(HttpRequest.BodyPublishers.ofString(json))
            .build();

        HttpResponse<String> response = client.send(
            request, HttpResponse.BodyHandlers.ofString()
        );

        System.out.println(response.statusCode() + ": " + response.body());
    }
}

reqwest 
use serde_json::json;

// Cargo.toml: reqwest = { version = "0.12", features = ["json"] }
//             tokio = { version = "1", features = ["full"] }
//             serde_json = "1"

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let webhook_url = "WEBHOOK_URL";

    let payload = json!({
        "event": "order.created",
        "data": {
            "id": 123,
            "customer": "jane@example.com",
            "total": 49.99
        }
    });

    let client = reqwest::Client::new();
    let resp = client
        .post(webhook_url)
        .json(&payload)
        .send()
        .await?;

    println!("Status: {}", resp.status());
    println!("Body: {}", resp.text().await?);
    Ok(())
}

Common Use Cases

Webhook provider testing
Use your OpenWhook URL as the callback in Stripe, GitHub, Shopify, or any service that sends webhooks. Inspect the exact payload they send before writing handler code.
Local development
Test your app's outgoing HTTP calls without needing the real endpoint. Point your app at the webhook URL and verify requests are shaped correctly.
Debugging integrations
When a webhook handler isn't working, use OpenWhook to see the raw request. Compare what you expect vs what actually arrives — headers, body encoding, content type.
CI/CD pipeline verification
Add a webhook call in your pipeline to confirm a stage completed. Check OpenWhook to verify the notification was sent with the right metadata.

API Reference

OpenWhook also exposes a simple REST API for managing sessions programmatically.

Create a new session
# Returns { id, url, createdAt }
curl -X POST WEBHOOK_BASE/api/sessions
Get session with all captured requests
curl WEBHOOK_BASE/api/sessions/YOUR-SESSION-ID
Delete a session
curl -X DELETE WEBHOOK_BASE/api/sessions/YOUR-SESSION-ID
